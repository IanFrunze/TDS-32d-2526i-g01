postGame

open override fun postGame(fileName: String, game: GameImpl)

Creates or updates the persisted game file with a full snapshot (headers and board).

Behavior:

  • If the target file does not exist or is empty, a new file is created and a full snapshot is written. The available pieces header is computed from game.players.

  • If the file exists, its headers are read first. The persisted board side must match game.board.side or an InvalidGameWriteException is thrown to avoid corruption. The persisted availablePieces header is filtered by the players present in game and the file is overwritten with the updated snapshot.

Parameters

fileName

target game file path or identifier

game

the game state to persist

Throws

IOException

when the underlying file I/O fails

InvalidGameWriteException

when an existing file contains an incompatible side
